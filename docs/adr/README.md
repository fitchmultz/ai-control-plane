# Architecture Decision Records

This directory is the home for Architecture Decision Records (ADRs).

## Purpose

ADRs capture durable architectural decisions that affect:

- support boundary
- operator UX
- domain ownership
- deployment shape
- validation and release discipline

## Naming

- `0001-short-kebab-case-title.md`
- `0002-another-decision.md`

## Status Values

- Proposed
- Accepted
- Superseded

## Starter Files

- `0000-template.md`
- `0001-host-first-docker-as-supported-surface.md`
