# Generated Artifacts and Evidence Policy

This repository intentionally does **not** commit generated runtime logs, handoff bundles, local agent state, or generated presentation exports.

## Local-Only Paths

The following paths are generated during local runs and are gitignored:

- `demo/logs/` — runtime logs, readiness evidence logs, and release-bundle output
- `demo/logs/pilot-closeout/` — generated pilot closeout bundles and bundle pointers
- `demo/backups/` — local backup artifacts
- `handoff-packet/` — exportable handoff bundles created for internal transfer
- `.ralph/` — local agent planning/cache state
- `docs/presentation/slides-external/*.png` — generated customer-facing slide image exports derived from the canonical external deck source

## Why These Are Not Committed

- Avoid committing sensitive operational data
- Keep repository diffs reviewable
- Prevent stale evidence from being mistaken as current certification data
- Keep clone size and history clean for public review
- Preserve a truthful boundary between authored source material and generated exports

## How to Regenerate Evidence Locally

```bash
# Generate a dated readiness proof pack
make readiness-evidence
make readiness-evidence-verify

# Generate a dated pilot closeout bundle from source documents plus readiness evidence
# Typical inputs now include the charter, measurable outcomes scorecard,
# acceptance memo, case study, and operator handoff checklist.
make pilot-closeout-bundle
make pilot-closeout-bundle-verify

# Optional: include production-like gate when a real secrets file exists
make readiness-evidence READINESS_INCLUDE_PRODUCTION=1 SECRETS_ENV_FILE=/etc/ai-control-plane/secrets.env

# Lower-level commands remain available when you need to troubleshoot one gate
make ci
make release-bundle
make release-bundle-verify

# Generate local benchmark evidence on demand
make up-offline
make performance-baseline PERFORMANCE_PROFILE=interactive
./scripts/acpctl.sh benchmark baseline --profile interactive --json

# Generate customer-facing presentation slide exports locally
cd docs/presentation
marp ai-control-plane-external-deck.md --images --output slides-external/
```

Artifacts generated by these commands are expected under `demo/logs/evidence/readiness-<TIMESTAMP>/`, `demo/logs/pilot-closeout/pilot-closeout-<TIMESTAMP>/`, `demo/logs/release-bundles/`, and `docs/presentation/slides-external/`.

## Local-Only Benchmark Outputs

Per-run benchmark results produced by:

- `make performance-baseline`
- `./scripts/acpctl.sh benchmark baseline --json`

are local-only runtime evidence and should not be committed as dated source artifacts.

Why:

- benchmark results go stale as the repository, images, and host change
- committed run captures can be mistaken for universal or current capacity proof
- buyer-safe publication here is the **methodology and sizing guide**, not frozen runtime numbers

The committed benchmark source material is:

- `docs/PERFORMANCE_BASELINE.md`

That file publishes the reproducible methodology, workload profiles, reference hardware tiers, interpretation guidance, and caveats. Operators and buyers should regenerate current run data locally when needed.

## Retention and Cleanup

```bash
# Retain newest evidence/bundle sets
make artifact-retention-check
make artifact-retention-apply

# Full local cleanup (destructive)
make clean-force
```
