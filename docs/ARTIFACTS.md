# Generated Artifacts and Evidence Policy

This repository intentionally does **not** commit generated runtime logs, handoff bundles, or local agent state.

## Local-Only Paths

The following paths are generated during local runs and are gitignored:

- `demo/logs/` — runtime logs, readiness evidence logs, and release-bundle output
- `demo/logs/pilot-closeout/` — generated pilot closeout bundles and bundle pointers
- `demo/backups/` — local backup artifacts
- `handoff-packet/` — exportable handoff bundles created for internal transfer
- `.ralph/` — local agent planning/cache state

## Why These Are Not Committed

- Avoid committing sensitive operational data
- Keep repository diffs reviewable
- Prevent stale evidence from being mistaken as current certification data
- Keep clone size and history clean for public review

## How to Regenerate Evidence Locally

```bash
# Generate a dated readiness proof pack
make readiness-evidence
make readiness-evidence-verify

# Generate a dated pilot closeout bundle from source documents plus readiness evidence
make pilot-closeout-bundle
make pilot-closeout-bundle-verify

# Optional: include production-like gate when a real secrets file exists
make readiness-evidence READINESS_INCLUDE_PRODUCTION=1 SECRETS_ENV_FILE=/etc/ai-control-plane/secrets.env

# Lower-level commands remain available when you need to troubleshoot one gate
make ci
make release-bundle
make release-bundle-verify
```

Artifacts generated by these commands are expected under `demo/logs/evidence/readiness-<TIMESTAMP>/`, `demo/logs/pilot-closeout/pilot-closeout-<TIMESTAMP>/`, and `demo/logs/release-bundles/`.

## Retention and Cleanup

```bash
# Retain newest evidence/bundle sets
make artifact-retention-check
make artifact-retention-apply

# Full local cleanup (destructive)
make clean-force
```
