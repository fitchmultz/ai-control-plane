Executive Summary
AI Control Plane · February 2026

The Problem

Organizations adopt AI through two channels requiring different controls:

Channel Risk Surface Current Gap
API Usage Apps/agents with API keys Shadow keys, uncontrolled spend, no attribution
Subscription/SaaS Developer tools via OAuth Unmanaged seats, data exfiltration, compliance gaps

Critical Insight: A strategy covering only API keys is incomplete. A strategy covering only SaaS is incomplete. Real-world programs must address both channels simultaneously.

Our Solution: Unified AI Control Plane

1. Gateway-Routed Control Plane

PREVENTIVE
  • AI Gateway chokepoint
  • Model allowlists
  • Content guardrails (PII, secrets, policy)
  • Budget enforcement
  • Real-time blocking
  • Full audit trail

2. Direct/Bypass Governance

DETECTIVE
  • OTEL telemetry
  • Compliance exports
  • Identity/workspace controls
  • Device posture
  • Investigation workflows

3. Evidence Pipeline

UNIFIED
  • Normalized SIEM feed
  • Unified schema
  • Detection rules
  • Response runbooks
  • Executive reporting

Capabilities: Enforce vs. Detect

Capability Gateway-Routed Mode Direct Subscription / Bypass
Content guardrails (PII, secrets) ✓ Real-time Detect + respond
Real-time blocking ✓ Full Detect + respond
Budget enforcement ✓ Yes Vendor-dependent
Usage attribution ✓ Yes ✓ Yes
Audit logging Gateway logs OTEL + compliance API

Honest Positioning: We enforce controls on gateway-routed traffic, apply detection and response on bypass paths, and maintain clear evidence trails for leadership and audit.
